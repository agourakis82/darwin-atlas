# Methods: Darwin Operator Symmetry Atlas

## Overview

This document provides formal definitions for the mathematical operators and algorithms used in the Darwin Operator Symmetry Atlas (DOSA). All methods are implemented in both Julia (reference implementation) and Sounio (high-performance kernels) with mandatory cross-validation between implementations.

---

## 1. Sequence Encoding

### 1.1 Nucleotide Alphabet

Sequences are encoded over the 4-letter DNA alphabet:

```
A = 0, C = 1, G = 2, T = 3
```

### 1.2 Complement Mapping

The Watson-Crick complement is defined as:

| Base | Complement |
|------|------------|
| A (0) | T (3) |
| C (1) | G (2) |
| G (2) | C (1) |
| T (3) | A (0) |

This can be computed as `complement(b) = b XOR 3` using bitwise operations.

## 2. Dihedral Group Operators

For a sequence `s` of length `n`, we define the **dihedral group D_n** acting on circular sequences. The group has order `2n` with generators:

### 2.1 Cyclic Shift Operator (S)

**Definition**: The cyclic shift by `k` positions:

```
S^k(s)[i] = s[(i + k) mod n]
```

**Properties**:
- `S^n = I` (identity)
- Generates cyclic subgroup `C_n` of order `n`

### 2.2 Reverse Operator (R)

**Definition**: Sequence reversal:

```
R(s)[i] = s[n - 1 - i]
```

**Properties**:
- `R^2 = I` (involution)
- `R * S * R = S^{-1}` (conjugation relation)

### 2.3 Complement Operator (K)

**Definition**: Base-by-base complement:

```
K(s)[i] = complement(s[i])
```

**Properties**:
- `K^2 = I` (involution)
- `K` commutes with `S` and `R`

### 2.4 Reverse-Complement Operator (RC)

**Definition**: Composition of reverse and complement:

```
RC = R o K = K o R
```

```
RC(s)[i] = complement(s[n - 1 - i])
```

**Properties**:
- `RC^2 = I`
- Biologically meaningful: represents the complementary DNA strand

### 2.5 Group Structure

The full symmetry group is D_n x Z_2, where:
- D_n is generated by `{S, R}`
- Z_2 is generated by `{K}`

For the Atlas, we primarily analyze the D_n action (with RC as a distinguished element).

## 3. Exact Symmetry Analysis

### 3.1 Fixed Points

A sequence `s` is **fixed** by an operator `g` if `g(s) = s`.

**R-fixed (Palindrome)**:
```
s is R-fixed iff s[i] = s[n-1-i] for all i
```

**RC-fixed**:
```
s is RC-fixed iff s[i] = complement(s[n-1-i]) for all i
```

### 3.2 Cyclic Period

**Definition**: The cyclic period `p` is the smallest positive integer such that `S^p(s) = s`:

```
p = min { k > 0 : S^k(s) = s }
```

**Algorithm**: Linear scan from `k=1` to `n`, checking equality.

**Complexity**: O(n^2) worst case, O(n) average case.

### 3.3 Orbit Size

**Definition**: The orbit of `s` under D_n is:

```
Orbit(s) = { g(s) : g in D_n }
```

**Orbit-Stabilizer Theorem**:
```
|Orbit(s)| = |D_n| / |Stab(s)|
```

where `|D_n| = 2n` and `|Stab(s)|` is the stabilizer size.

**Computation**:
```
rot_sym = n / cyclic_period(s)    # Rotational symmetries
ref_factor = has_reflective_symmetry(s) ? 2 : 1
orbit_size = 2n / (rot_sym * ref_factor)
```

**Reflective Symmetry Check**: A sequence has reflective symmetry if there exists `k` such that `R(s) = S^k(s)`:
```
for k in 0..n-1:
    if R(s) == S^k(s): return true
return false
```

### 3.4 Orbit Ratio

**Definition**: Normalized orbit size:

```
orbit_ratio = |Orbit(s)| / |D_n| = |Orbit(s)| / (2n)
```

**Range**: `0.25 <= orbit_ratio <= 1.0`
- `orbit_ratio = 1.0`: No symmetry (generic sequence)
- `orbit_ratio = 0.5`: One type of symmetry
- `orbit_ratio = 0.25`: Maximal symmetry

## 4. Approximate Symmetry Metric

### 4.1 Hamming Distance

**Definition**: Number of positions where sequences differ:

```
hamming(a, b) = sum_{i=0}^{n-1} (a[i] != b[i] ? 1 : 0)
```

### 4.2 Dihedral Distance (d_min)

**Definition**: Minimum distance to any dihedral transform of the sequence:

```
d_min(s) = min { hamming(s, g(s)) : g in D_n, g != I }
```

**Algorithm**:
```
d_min = infinity
for k in 1..n-1:
    d_min = min(d_min, hamming(s, S^k(s)))
d_min = min(d_min, hamming(s, R(s)))
for k in 0..n-1:
    d_min = min(d_min, hamming(s, R(S^k(s))))
return d_min
```

**Complexity**: O(n^2)

### 4.3 Normalized d_min

**Definition**: Length-normalized distance:

```
d_min_normalized = d_min / n
```

**Range**: `0.0 <= d_min_normalized <= 1.0`
- `d_min_normalized = 0.0`: Sequence is fixed by some non-identity element
- Higher values indicate sequences far from any symmetric configuration

## 5. Dicyclic Lift (Quaternion Verification)

### 5.1 Dicyclic Group Dic_n

The dicyclic group Dic_n is a double cover of D_n with order `4n`. It is defined by:

```
<a, x | a^{2n} = 1, x^2 = a^n, x^{-1}ax = a^{-1}>
```

### 5.2 Projection to D_n

The projection pi: Dic_n -> D_n is:
- `pi(a^k) = S^k` (rotations)
- `pi(a^k * x) = S^k * R` (reflections)

### 5.3 Verification

We verify that Dic_n is a valid double cover by checking:
1. `|Dic_n| = 2 * |D_n|`
2. `ker(pi) = {1, a^n}` (two elements)
3. Group relations are satisfied

## 6. Null Model Validation

### 6.1 GC-Preserving Shuffle

**Definition**: Random permutation of a sequence that preserves nucleotide composition.

**Algorithm** (Fisher-Yates shuffle):
```
shuffled = copy(seq)
for i in (n-1)..1:
    j = random(0..i)
    swap(shuffled[i], shuffled[j])
return shuffled
```

### 6.2 Statistical Test

For each sequence:
1. Compute observed `orbit_ratio`
2. Generate `N` shuffled sequences (N=100)
3. Compute `orbit_ratio` for each shuffle
4. Calculate z-score:
   ```
   z = (observed - mean_shuffle) / std_shuffle
   ```
5. Compute two-tailed p-value from standard normal distribution

### 6.3 Interpretation

- Under null hypothesis (no real symmetry): ~5% significant at p < 0.05
- Significantly higher rates indicate genuine biological symmetry patterns

## 7. Sliding Window Analysis

### 7.1 Circular Window Extraction

For circular genomes, window extraction handles wraparound:

```
function extract_window(seq, start, window_size):
    n = length(seq)
    start_mod = start mod n
    end_pos = start_mod + window_size

    if end_pos <= n:
        return seq[start_mod : end_pos]
    else:
        # Wraparound
        return seq[start_mod : n] ++ seq[0 : end_pos - n]
```

### 7.2 Window Positions

For non-overlapping windows:
```
positions = [0, window_size, 2*window_size, ...]
```

### 7.3 Window Sizes

Standard analysis uses: 100, 500, 1000, 5000, 10000 bp

## 8. Cross-Validation Protocol

### 8.1 Requirement

Julia and Sounio implementations must produce **identical** results:
- Discrete values: exact match required
- Floating point: tolerance of 10^{-12}

### 8.2 Procedure

1. Generate test sequences (random and special cases)
2. Compute all metrics in both implementations
3. Compare results element-by-element
4. **Any divergence is a blocking bug**
